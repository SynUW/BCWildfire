# 逐日驱动因素透视表更新说明

## 更新概述

根据用户需求，修改了 `daily_driver_analysis.py` 中的透视表生成逻辑，现在透视表包含燃烧像元和对照像元的**所有统计信息**，而不仅仅是差异值。

## 主要修改

### 1. 统计信息包含内容

**燃烧像元统计量（8种）：**
- `burned_count` - 燃烧像元数量
- `burned_mean` - 燃烧像元均值
- `burned_median` - 燃烧像元中值  
- `burned_std` - 燃烧像元标准差
- `burned_min` - 燃烧像元最小值
- `burned_max` - 燃烧像元最大值
- `burned_q25` - 燃烧像元下四分位数（25%）
- `burned_q75` - 燃烧像元上四分位数（75%）

**对照像元统计量（8种）：**
- `control_count` - 对照像元数量
- `control_mean` - 对照像元均值
- `control_median` - 对照像元中值
- `control_std` - 对照像元标准差
- `control_min` - 对照像元最小值
- `control_max` - 对照像元最大值
- `control_q25` - 对照像元下四分位数（25%）
- `control_q75` - 对照像元上四分位数（75%）

**总计：16种统计量**

### 2. 输出文件结构

**唯一透视表：** `daily_driver_pivot_10days.csv`
- 行：特征名_统计量类型（如：`ERA5_Temperature_band_1_burned_mean`）
- 列：燃烧前天数（1, 2, 3, ..., 10）
- 值：对应的统计量数值

### 3. 透视表格式示例

```
days_before                                1          2          3         ...
ERA5_Temperature_band_1_burned_mean    101.36     101.12     98.85     ...
ERA5_Temperature_band_1_burned_median  101.95     100.34     99.24     ...  
ERA5_Temperature_band_1_burned_std       9.58      10.76     10.18     ...
ERA5_Temperature_band_1_control_mean    92.86      95.36     95.09     ...
ERA5_Temperature_band_1_control_median  92.71      95.03     95.08     ...
ERA5_Temperature_band_1_control_std      7.99       7.35      8.49     ...
...
```

## 使用方法

### 运行分析

```python
from daily_driver_analysis import DailyDriverAnalyzer

analyzer = DailyDriverAnalyzer(
    data_dir="数据目录路径",
    output_dir="输出目录路径", 
    firms_dir="FIRMS目录路径",
    days_before=10
)

success = analyzer.run_analysis()
```

### 读取透视表结果

```python
import pandas as pd

# 读取唯一透视表
pivot_table = pd.read_csv('daily_driver_pivot_10days.csv', index_col=0)

# 筛选特定统计量
burned_mean_rows = pivot_table[pivot_table.index.str.contains('_burned_mean')]
control_mean_rows = pivot_table[pivot_table.index.str.contains('_control_mean')]
```

## 关键特点

1. **完整统计信息**：包含燃烧像元和对照像元的所有重要统计量
2. **分别显示**：燃烧像元和对照像元的统计值分别列出，不是差异值
3. **单文件输出**：所有统计信息集中在一个综合透视表文件中
4. **时间维度**：每个统计量按燃烧前天数（1-10天）展示变化趋势
5. **驱动因素覆盖**：包含所有动态驱动因素的所有波段

## 数据解读

- **燃烧像元统计**：反映实际发生燃烧的像元在燃烧前各天的驱动因素特征
- **对照像元统计**：反映未燃烧像元在相同时期的驱动因素特征  
- **对比分析**：通过比较燃烧像元和对照像元的统计值，可以识别燃烧风险的关键指标
- **时间趋势**：通过查看燃烧前1-10天的变化，可以分析燃烧前驱动因素的演变模式

## 科研应用

这种透视表格式特别适合：
1. **风险因子识别**：比较燃烧像元和对照像元在各驱动因素上的差异
2. **预警时间窗口分析**：分析燃烧前几天的变化最显著
3. **机器学习特征工程**：为模型训练提供结构化的特征数据
4. **可视化分析**：便于制作热图等可视化图表展示风险模式
