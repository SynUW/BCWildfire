# 野火点燃像素前10天逐日驱动因素分布分析器使用说明

## 功能概述

这个新的分析器(`DailyDriverAnalyzer`)专门用于统计野火点燃像素前10天的逐日驱动因素分布状况，满足您对逐日、逐驱动因素统计分析的需求。

## 核心功能

### 1. 逐日统计分析
- **燃烧前第1天**: 所有燃烧像元在燃烧前1天的每个驱动因素统计
- **燃烧前第2天**: 所有燃烧像元在燃烧前2天的每个驱动因素统计
- **...**
- **燃烧前第10天**: 所有燃烧像元在燃烧前10天的每个驱动因素统计

### 2. 对照组统计
对每个燃烧事件，自动选择相同数量的未燃烧像元作为对照组，在相同日期进行统计，确保对比的公平性。

### 3. 统计指标
对每日每个驱动因素计算以下统计量：
- **均值** (`mean`)
- **中值** (`median`) 
- **标准差** (`std`)
- **最小值** (`min`)
- **最大值** (`max`)
- **25%分位数** (`q25`)
- **75%分位数** (`q75`)
- **样本数量** (`count`)

## 输出结果

### 1. 详细统计文件
`daily_driver_analysis_10days.csv`
包含每个燃烧事件、每天、每个驱动因素的详细统计结果：

```csv
event_id,burn_date,target_date,days_before,driver_name,band_index,feature_name,
burned_count,burned_mean,burned_median,burned_std,burned_min,burned_max,burned_q25,burned_q75,
control_count,control_mean,control_median,control_std,control_min,control_max,control_q25,control_q75,
mean_difference,median_difference,std_ratio
```

### 2. 汇总分析报告
`daily_driver_summary_10days.txt`
包含：
- 按燃烧前天数的统计汇总
- 按驱动因素的统计汇总
- 平均差异最大的特征-天数组合排序

### 3. 透视表分析
`daily_driver_pivot_10days.csv`
以驱动因素为行、燃烧前天数为列的透视表，便于观察时间趋势。

## 使用方法

### 基本用法
```python
from daily_driver_analysis import DailyDriverAnalyzer

# 创建分析器
analyzer = DailyDriverAnalyzer(
    data_dir="/path/to/data",           # 驱动因素数据根目录
    output_dir="/path/to/output",       # 输出目录
    firms_dir="/path/to/firms",         # FIRMS检测数据目录
    days_before=10                      # 分析燃烧前天数
)

# 运行分析
success = analyzer.run_analysis()
```

### 自定义分析天数
```python
# 分析燃烧前5天
analyzer = DailyDriverAnalyzer(
    data_dir=data_dir,
    output_dir=output_dir,
    firms_dir=firms_dir,
    days_before=5
)
```

## 分析流程

### 1. 燃烧事件识别
- 扫描FIRMS数据，识别所有燃烧事件
- 记录每个事件的日期和燃烧像元位置
- 构建所有有效像元的集合

### 2. 对照组选择
- 对每个燃烧事件，选择相同数量的未燃烧像元
- 排除该日期及之后30天内燃烧过的像元
- 随机采样保证代表性

### 3. 逐日数据提取
- 对每个燃烧事件的燃烧前1-10天
- 查找每个驱动因素在目标日期的数据文件
- 提取燃烧像元和对照像元的数值

### 4. 统计计算
- 计算燃烧组和对照组的各项统计指标
- 计算两组间的差异
- 过滤无效值和极值

### 5. 结果汇总
- 生成详细的CSV文件
- 创建汇总分析报告
- 制作透视表便于分析

## 注意事项

1. **数据完整性**: 确保FIRMS数据和驱动因素数据的时间覆盖一致
2. **计算时间**: 完整分析可能需要较长时间，建议先用少量数据测试
3. **存储空间**: 结果文件可能较大，确保有足够存储空间
4. **参数调整**: 可根据具体需求调整分析天数

## 常见问题

**Q: 为什么某些日期没有数据？**
A: 可能是驱动因素数据缺失，程序会自动跳过这些日期。

**Q: 如何处理数据不平衡？**
A: 程序自动为每个燃烧事件选择相同数量的对照像元，保证平衡性。

**Q: 能否分析超过10天？**
A: 可以，通过修改`days_before`参数即可。
